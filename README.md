# Calibrated Cross-Sectional Return Forecaster (CSRF)

A research pipeline that applies conformal prediction to ML-based stock return forecasting and tests whether model uncertainty (interval width) is itself a cross-sectional return signal.

## Quick Start

```bash
# Create and activate the virtual environment (do this once)
python -m venv .venv

# Windows PowerShell
.\.venv\Scripts\Activate.ps1
# macOS / Linux
source .venv/bin/activate

# Install dependencies
pip install -r requirements.txt
```

Run the full pipeline phase by phase (venv must be active):

```bash
python scripts/01_build_data.py        # Phase 1: preprocess GKX data
python scripts/02_train_model.py       # Phase 2: train NN, verify R²
python scripts/03_conformal_calibrate.py  # Phase 3: fit SPCI, validate coverage
python scripts/04_width_analysis.py   # Phase 4: width-signal tests
python scripts/05_portfolio_eval.py   # Phase 5: portfolio construction
python scripts/06_robustness.py       # Phase 6: robustness checks
```

For a fast end-to-end smoke test on CPU (runs in minutes):

```bash
# Edit config.yaml: set fast_dev: true
python scripts/01_build_data.py --fast-dev
python scripts/02_train_model.py --fast-dev
# etc.
```

Read [LEARNING.md](LEARNING.md) — it explains the theory, every design decision, and how the codebase is structured. Start there before reading any code.

## Project Structure

```
src/data/       — GKX data loading, rank normalization, lag enforcement, splits
src/models/     — ReturnPredictor NN, training loop, evaluation metrics
src/conformal/  — SPCI conformal calibration, fallback, coverage validation
src/analysis/   — Width-signal tests, portfolio construction
src/utils/      — Config dataclass, I/O helpers
scripts/        — End-to-end pipeline scripts (run in order)
results/        — CSV tables and figures (git-ignored, generated by scripts)
data/           — Raw and processed data (git-ignored)
```
